# Инструкция: Склад Чапан

## Роли и кто что видит

| Роль | Раздел |
|---|---|
| Менеджер | Заказы → создаёт заказы, видит наличие прямо при оформлении |
| Швея / цех | Цех, Готово → подтверждает отправку товара накладной |
| Заведующий складом | Склад → принимает товар, управляет остатками, отгружает клиенту |

---

## Статусная цепочка заказа

```
Новый → Подтверждён → В производстве → Готово → На складе → Отгружен → Завершён
```

- **Готово** — цех закончил работу, товар физически готов
- **На складе** — заведующий принял, ответственность перешла к складу
- **Отгружен** — товар отправлен клиенту
- **Завершён** — сделка закрыта, заказ в архиве

Вернуть заказ из «На складе» обратно в «Готово» можно только с обязательной причиной.

## Часть 1. Менеджер — Создание Заказа

### Наличие товара при оформлении

При заполнении позиций заказа под каждым товаром появляется цветной бейдж:

- 🟢 **В наличии: N шт.** — товар есть, можно оформлять
- 🟡 **В наличии: N шт. (мало)** — остаток ниже минимального порога
- 🔴 **Нет на складе** — остаток нулевой

Бейдж учитывает именно тот вариант, который выбран: модель + цвет + размер + пол. Если цвет или размер не выбраны, система ищет по названию товара.

Наличие обновляется автоматически и учитывает уже зарезервированные товары под другие заказы.

### Флаг «Требуется накладная»

На каждом заказе есть флаг **Требуется накладная: Да / Нет**.

- **Нет** — заказ можно передать на склад напрямую, без оформления накладной
- **Да** — передать на склад без подтверждённой накладной нельзя; система заблокирует перевод

Флаг можно переключить в карточке заказа.

---

## Часть 2. Швея / Цех — Работа с Накладными

### Где смотреть накладные

Раздел **Накладные** в меню. Там видны все накладные с фильтрами:
- Все
- Ожидают (нужно подтверждение)
- Подтверждены
- Отклонены

Вверху списка — **Сверка-бар**: показывает сколько накладных ждёт цех, сколько ждёт склад, и сколько «зависших» (одна сторона подтвердила, другая нет больше 24 часов).

### Подтвердить накладную (со стороны цеха)

1. Открыть нужную накладную
2. Нажать **«Отправлено»**

После этого накладная переходит в состояние «Цех подтвердил, ожидает склад».

### Отклонить накладную

Если состав или данные неверные — нажать кнопку **«Отклонить»** и указать причину. Заказ вернётся в работу.

### Бейдж «Зависла»

Если напротив накладной стоит красный бейдж **«Зависла»** — одна из сторон подтвердила больше суток назад, но вторая сторона ещё не подтвердила. Нужно уточнить у второй стороны.

## Часть 3. Заведующий Складом — Основная Работа

### Вкладки в разделе «Склад»

| Вкладка | Что показывает |
|---|---|
| Входящие | Накладные, ожидающие приёмки со стороны склада |
| На складе | Заказы, принятые и хранящиеся на складе |
| К отгрузке | Заказы, готовые к отправке клиенту (оплачены) |
| Отгруженные | История отгруженных заказов |
| Остатки | Текущие складские остатки по всем позициям |
| Движения | Журнал всех приходов, расходов, корректировок |
| Алерты | Позиции с низким остатком |
| Каталог | Справочник товаров (canonical WMS) |

---

### 3.1. Приёмка товара по накладной

1. Перейти на вкладку **Входящие**
2. Найти нужную накладную. Если одна сторона уже подтвердила — это видно в колонке «Статус цеха»:
   - ✅ Цех отправил
   - ⏳ Ожидает цех
3. Нажать **«Открыть»**, просмотреть состав
4. Нажать **«Принято»** — накладная подтверждена с обеих сторон, заказы автоматически переходят в статус «На складе»

Если состав не совпадает:
- Нажать **«Отклонить»** и указать причину
- Заказы останутся в «Готово», цех будет уведомлён

### 3.2. Возврат заказа в «Готово»

Если заказ уже принят, но нужно его вернуть (ошибка в составе, несоответствие количества):

1. Открыть заказ из вкладки **На складе**
2. Нажать **«Вернуть в «Готово»»**
3. Указать причину (обязательно)

Заказ вернётся в «Готово», в истории останется запись. Простые резервы на складе автоматически освобождаются.

### 3.3. Отгрузка клиенту

Перейти на вкладку **К отгрузке**. Там отображаются только заказы, которые:
- находятся в статусе «На складе»
- **оплачены полностью**

Неоплаченные заказы отгрузить нельзя — система заблокирует с сообщением об остатке долга.

Нажать **«Отгрузить»** на нужном заказе. После отгрузки:
- заказ переходит в «Отгружен»
- складские резервы и остатки обновляются автоматически

---

### 3.4. Управление остатками

#### Просмотр

Вкладка **Остатки**. Колонки:
- **Вариант** — модель + атрибуты (цвет, пол, размер)
- **Кол-во** — физический остаток
- **Мин.** — минимальный порог (ниже него — алерт)
- Поиск и фильтры по категории, низкому остатку

#### Добавить позицию вручную

Кнопка **«+ Добавить»** в тулбаре остатков. Заполнить:
- Название товара
- Цвет, пол, размер (вариант)
- Количество
- Минимальный порог

#### Загрузить стартовые остатки из Excel/CSV

Кнопка **«Импорт остатков»**. Три экрана:

1. **Загрузка** — вставить CSV-текст или файл. Ожидаемые колонки:
   ```
   наименование, цвет, пол, размер, остаток, себестоимость
   ```
   Порядок колонок гибкий, система распознаёт по заголовку.

2. **Предпросмотр** — показывает первые строки, можно проверить до загрузки

3. **Результат** — сколько позиций создано, обновлено, пропущено, с ошибками

Если позиция с таким вариантом уже есть — остаток прибавляется, дубль не создаётся.

#### Добавить движение вручную

Кнопка **«+ Движение»**:

| Тип | Когда |
|---|---|
| Приход | Получили товар от поставщика / нашли лишнее |
| Расход | Выдали товар не через заказ |
| Возврат | Товар вернули от клиента |
| Корректировка | Инвентаризация, пересчёт |
| Списание | Брак, утеря |

**Важно:** расход и списание нельзя провести, если свободного остатка не хватает. Корректировка — единственный тип, который может обходить резерв, но не может опустить общий остаток ниже нуля.

---

### 3.5. Журнал движений

Вкладка **Движения**. Для каждой записи видно:
- Тип движения
- Количество (с указанием «до → после»)
- Источник (Заказ / Нач. остаток / Сделка)
- Автор и дата

---

### 3.6. Алерты о низком остатке

Вкладка **Алерты** — автоматически создаётся, когда остаток позиции опускается ниже минимального порога. После пополнения алерт разрешается автоматически.

## Часть 4. Накладная — Детали

### Что такое накладная

Накладная — это электронный акт передачи товара между цехом и складом. Она нужна для внутренней фиксации ответственности, не для клиента.

Накладная содержит:
- Номер и дату
- Рейс / маршрут
- Список позиций (товар, пол, размер, цвет, кол-во, цена)
- Статус подтверждений с каждой стороны

### Редактирование накладной

Накладную можно отредактировать прямо в интерфейсе — нажать **«Редактировать»** в окне просмотра. Доступно изменение:
- Количества
- Цены (клиентской и внутренней)
- Прочих полей строк

Изменения сохраняются кнопкой **«Сохранить»**. При закрытии с несохранёнными правками система спросит подтверждение.

### Внутренняя цена vs. клиентская цена

В накладной два ценовых поля:
- **Цена (unitPrice)** — цена для клиента
- **Внутр. цена (warehouseUnitPrice)** — себестоимость / внутренняя цена цеха

Колонка «Внутр. цена» появляется в таблице только если хотя бы одна строка накладной имеет это поле заполненным.

### Скачать накладную

Кнопка **«Скачать»** в окне просмотра — выгружает накладную в формате `.xlsx`.

## Часть 5. Типичные Сценарии

### Сценарий А: Заказ без накладной

1. Менеджер создаёт заказ, `requiresInvoice = Нет`
2. Заказ проходит производство, статус «Готово»
3. Заведующий склада принимает заказ напрямую → статус «На складе»
4. После оплаты → «Отгрузить» → статус «Отгружен»

### Сценарий Б: Заказ с обязательной накладной

1. Менеджер создаёт заказ, `requiresInvoice = Да`
2. Заказ в статусе «Готово»
3. Создаётся накладная с нужными позициями
4. Швея подтверждает: **«Отправлено»**
5. Заведующий проверяет состав и подтверждает: **«Принято»**
6. Оба подтвердили → заказ автоматически переходит в «На складе»
7. После оплаты → отгрузка

*Попытка перевести заказ в «На складе» вручную без накладной заблокируется с сообщением.*

### Сценарий В: Расхождение при приёмке

1. Заведующий открывает накладную во вкладке «Входящие»
2. Видит несоответствие (другой цвет, меньше штук)
3. Нажимает **«Отклонить»**, пишет причину: «Не совпадает цвет по позиции 3»
4. Цех видит отклонение, исправляет, создаёт новую накладную

### Сценарий Г: Возврат от клиента

1. Клиент вернул товар
2. Заведующий на вкладке «Остатки» → **«+ Движение»** → тип **«Возврат»**
3. Указывает позицию, количество, причину
4. Остаток увеличивается, в журнале движений появляется запись

### Сценарий Д: Инвентаризация

1. Пересчитали физический остаток — обнаружили расхождение
2. **«+ Движение»** → тип **«Корректировка»**
3. Ввести количество (со знаком «–» если списать лишнее)
4. Система не даст уйти ниже нуля по общему остатку

## Часть 6. Частые Вопросы

**Почему нельзя отгрузить заказ?**
Проверьте: заказ должен быть в статусе «На складе» и полностью оплачен. Неоплаченный заказ система не даёт отгрузить.

**Почему нельзя перевести заказ на склад?**
Если у заказа флаг «Требуется накладная: Да» — нужна накладная с двусторонним подтверждением. Без неё перевод заблокирован.

**Почему наличие в заказе показывает меньше, чем фактически на складе?**
Часть остатка зарезервирована под другие заказы. Система показывает только **свободный** остаток (общий минус зарезервированный).

**Что значит бейдж «Зависла» на накладной?**
Одна из сторон подтвердила накладную, а вторая не подтвердила больше 24 часов. Нужно напомнить второй стороне.

**Можно ли отредактировать уже подтверждённую накладную?**
Да, документ накладной редактируется независимо от её статуса. Изменения сохраняются в системе, история действий доступна через журнал активности заказа.

**Как загрузить начальные остатки из Excel?**
Вкладка «Остатки» → кнопка **«Импорт остатков»**. Скопировать данные из Excel в формате CSV (наименование, цвет, пол, размер, остаток). Система покажет предпросмотр перед загрузкой.
