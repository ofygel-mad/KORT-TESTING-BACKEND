# #9 SPA-Reset Bug — Воспроизведение и тестирование

## Описание бага
**Было:** При полной перезагрузке SPA (F5, Ctrl+R) во время заполнения новой формы заказа все данные в форме теряются.

**Ожидается:** Autosave через localStorage должен восстанавливать черновик при перезагрузке страницы.

## Реализация Autosave
- **Файл:** `src/pages/workzone/chapan/orders/ChapanNewOrder.tsx`
- **Ключ хранилища:** `chapan_new_order_draft_v1`
- **Дебаунс:** 800 мс
- **Логика:** Не сохраняет пустые черновики (требуется имя клиента ИЛИ номер телефона ИЛИ товар)
- **UI:** Баннер "Восстановлен незавершённый черновик" с кнопкой очистки

## Инструкция по воспроизведению

### Сценарий 1: Базовая проверка autosave
1. Откройте страницу "Новый заказ"
2. Заполните поля:
   - Имя клиента: "Тестовый клиент"
   - Номер телефона: "+7 (700) 123-45-67"
   - Город: "Алматы"
3. Дождитесь 1 сек (дебаунс 800 мс + буфер)
4. **Полная перезагрузка:** Нажмите Ctrl+Shift+R (или F5, или cmd+shift+r на Mac)
5. ✅ **Ожидаемый результат:**
   - Появится баннер "Восстановлен незавершённый черновик"
   - Поля восстановлены с введёнными значениями

### Сценарий 2: Проверка с товарами
1. Откройте "Новый заказ"
2. Добавьте товары:
   - Товар 1: "Рубашка", Цвет: "Синий", Размер: "42", Кол-во: "2", Цена: "3000"
   - Товар 2: "Платье", Цвет: "Красный", Размер: "44", Кол-во: "1", Цена: "5000"
3. Дождитесь 1 сек
4. **Перезагрузка:** F5
5. ✅ **Ожидаемый результат:**
   - Баннер восстановления
   - Все товары восстановлены с точными значениями (цвет, размер, кол-во, цена)

### Сценарий 3: Проверка с платежами
1. Откройте "Новый заказ"
2. Заполните:
   - Имя: "Клиент"
   - Добавьте товар: "Рубашка", размер "42", кол-во "1", цена "3000"
   - Способ оплаты: "Смешанный"
   - Наличные: "1000"
   - Kaspi QR: "2000"
3. Дождитесь 1 сек
4. **Перезагрузка:** Ctrl+Shift+R
5. ✅ **Ожидаемый результат:**
   - Способ оплаты "Смешанный" выбран
   - Суммы: 1000 (наличные) + 2000 (QR) восстановлены

### Сценарий 4: Пустая форма НЕ сохраняется
1. Откройте "Новый заказ"
2. Ничего не заполняйте
3. Дождитесь 1 сек
4. **Перезагрузка:** F5
5. ✅ **Ожидаемый результат:**
   - Баннер восстановления НЕ появляется
   - Форма остаётся пустой (как новая)

### Сценарий 5: Очистка черновика
1. Откройте "Новый заказ"
2. Заполните: Имя "Клиент"
3. Дождитесь 1 сек
4. Появится баннер "Восстановлен незавершённый черновик"
5. Нажмите кнопку очистки (крестик) в баннере
6. Дождитесь 1 сек
7. **Перезагрузка:** F5
8. ✅ **Ожидаемый результат:**
   - Баннер НЕ появляется
   - Форма пустая

### Сценарий 6: Отправка формы очищает черновик
1. Откройте "Новый заказ"
2. Заполните минимум (Имя + товар)
3. Нажмите "Создать заказ"
4. ⏳ Дождитесь успешного создания (без ошибок)
5. **Перезагрузка страницы:** F5
6. ✅ **Ожидаемый результат:**
   - Баннер восстановления НЕ появляется
   - Форма пустая (черновик был удален после отправки)

1. Откройте DevTools (F12)

3. Найдите хост вашего приложения
4. Поищите ключ `chapan_new_order_draft_v1`
5. Содержимое должно обновляться при заполнении формы (с дебаунсом 800 мс)

## Известные ограничения
- Загруженные файлы (receiptFileNames) НЕ сохраняются (реализация загрузки еще в процессе)
- Черновик сохраняется только в localStorage текущего браузера (не синхронизируется между вкладками)
- Старые черновики (версия < v1) удаляются при обновлении схемы
